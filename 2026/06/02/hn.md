## [The newest Instagram "exploit" is the goofiest I've seen](https://www.0xsid.com/blog/meta-account-takeover-fiasco) — [472 comments](https://news.ycombinator.com/item?id=48359102) | 2118 points

**Article:** A critical Instagram security vulnerability allowed attackers to hijack accounts by exploiting Meta's support AI system, requiring only a username and spoofed location data to convince the bot to reset account credentials to an attacker-controlled email. The flaw bypassed two-factor authentication entirely and enabled high-profile account takeovers including the Obama White House account before Meta patched it.

**Discussion:** The discussion focuses on how account recovery is always the weakest link in any authentication system, noting that 2FA can be rendered meaningless when low-level support staff (or AI bots) can bypass it. The debate grapples with the fundamental design trade-off between preventing account lockout (fail-safe) and preventing unauthorized access (fail-secure), with proposed solutions ranging from time delays to in-person authentication.

**Article:** A security vulnerability was disclosed affecting 31 npm packages under the @redhat-cloud-services/ scope, where malicious versions were published to these packages. Users were advised to avoid the compromised versions and update to patched releases.

**Discussion:** The thread debates supply chain security in the JavaScript/npm ecosystem, with significant discussion around "cooldown" proposals that would delay installation of newly published packages by 1–7 days to allow security scanners time to detect malicious code. A cultural divide emerges between embedded/traditional software developers who express shock at npm's deep dependency graphs and JavaScript developers who see it as necessary for velocity. The consensus is that cooldowns offer modest protection, but lasting solutions require broader ecosystem changes including better publishing safeguards and dependency vendoring.
